## Rules
- **Docs first, AI second**: Read official docs, error output, source code, or your own notes before asking AI.
- **Thirty-minute struggle rule**: When stuck, spend focused time reproducing the issue, isolating the layer, reading logs, and writing a hypothesis.
- **No copy-paste coding**: Use AI for hints, explanations, and review. Type and understand the final code yourself.
- **Explain before building**: Write the data model, request flow, UI state, failure cases, and tests before implementing meaningful features.
- **Bug journal**: Every bug should produce a reusable note: symptom, cause, fix, prevention, and lesson.
- **Ship monthly**: Every month must produce something runnable or demonstrable.
